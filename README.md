# Tomo Island

A **Tomodachi Life–style virtual life game** that runs entirely in the browser.
Build little characters (Tomodachis), drop them on an island, and they chat
with each other and with you. Their speech is generated by an on-device
LLM (**Gemma 4** via MediaPipe Tasks GenAI) and spoken aloud using the
**Web Speech API** with a pitch-shifted "Mii voice."

Everything is procedural — sprites, audio, music — so the entire shipped build
is well under 1 MB. The LLM model is the only large download, and it's
streamed once from the Hugging Face CDN and then cached by the browser.

```
+-----------------------------+
|  Vite + TypeScript 5 (strict)|
+-----------------------------+
|  PixiJS v8 (WebGPU / WebGL) |
+-----------------------------+
|  MediaPipe Tasks GenAI      |  ← lazy-loaded, ~1-2 GB on first chat
|  Web Speech API (TTS)       |  ← built-in, no dependency
|  IndexedDB (saves)          |  ← via `idb`
+-----------------------------+
```

## Quick start (local)

```bash
cd web-rpg
npm install
npm run dev
```

Open <http://localhost:5173>. Click **✨ New Island** to start.

```bash
npm run build      # type-check + production build → dist/
npm run preview    # serve dist/ on :4173
```

> Requires **Node 20+**.

## Browser requirements

| Feature       | Required for                      | Browsers                            |
|---------------|-----------------------------------|-------------------------------------|
| WebGPU *or* WebGL2 | PixiJS rendering              | Chrome 113+, Edge 113+, Firefox 121+ (limited), Safari 17+ |
| SpeechSynthesis     | Tomodachi voice lines        | Chrome, Edge, Firefox, Safari (varies) |
| IndexedDB           | Save / load persistence      | All modern browsers                  |
| SharedArrayBuffer / cross-origin isolation | LLM inference (MediaPipe) | Required: Vercel headers send `COOP: same-origin` + `COEP: credentialless` automatically |

> **First-time LLM download:** ~1–2 GB. The browser caches the model file via
> HTTP, so subsequent visits are instant.

## Deploy to Vercel (testing)

The repo includes a `vercel.json` with the required cross-origin headers and
SPA rewrite. From the repo root:

```bash
cd web-rpg
npx vercel        # first time: log in, accept defaults
npx vercel --prod # promote to production
```

Vercel will detect Vite, build with `npm run build`, and serve `dist/`.
The game loads at the project URL.

> If you fork or rename the project, Vercel reads `vercel.json` from
> `web-rpg/` automatically (when `web-rpg/` is the project root).

## Deploy to itch.io (final)

itch.io is the same static `dist/` output. Build, zip, upload:

```bash
cd web-rpg
npm run build:itchio
# → tomo-island-itchio.zip
```

On itch.io:

1. **Create new project** → kind: **HTML5**.
2. **Upload** the zip.
3. Set *"This file will be played in the browser"* = **ON**.
4. Set *Embed options* → viewport dimensions: **960×540** (integer-scaled 2× from 480×270 base).
5. Click **Save & View Page**.

The zip is ~600 KB without the LLM model (which streams at runtime).

## How it works

### Procedural art

Every sprite, tile, and animation frame is drawn at runtime from a tiny
string-grid "pixel sheet" system in `src/render/pixel.ts`. There are no PNGs
or audio files in the build. Sprites are composed from layered "parts"
(`head`, `eyes`, `mouth`, `hair`, `body`, `outfit`, `accessory`) with
palette tints applied per Tomodachi.

### Hybrid LLM

- **Barks (scripted, free, instant):** every few seconds an idle Tomodachi
  says a random line from `src/data/barks.ts`, filtered by mood and
  current activity. No model call.
- **Dialogue (LLM):** when the player clicks **Talk**, the LLM is loaded
  lazily on first use, then queried with a system prompt that includes the
  Tomodachi's personality (OCEAN traits), mood, needs, recent events, and
  the player's typed message. Tokens stream into the speech bubble, and
  TTS fires when the reply finishes.

### Mii voice

TTS uses `SpeechSynthesisUtterance` with `pitch = 1.2–1.7` and
`rate = 1.05–1.15` per Tomodachi so they each have a distinct "voice."

### Time / day-night

Real-time anchored: 1 real minute = 1 in-game minute. Anchored at game
start (no drift, no accumulated error). The day/night tint comes from
`src/render/effects.ts` and shifts the canvas color grade.

### Save / load

Auto-save every 30 s to **localStorage** and **IndexedDB** (`autosave`
slot). Manual save via the **💾 Save** button. **Continue** on the title
screen reloads the most recent state.

## Configuration knobs

| File                          | What                                    | Default                                    |
|-------------------------------|-----------------------------------------|--------------------------------------------|
| `src/game/time.ts`            | `TIME_SCALE` (real min → game min)      | `1`                                        |
| `src/llm/loader.ts`           | `MODEL_URL` + fallbacks                 | `gemma-4-E2B-it-int4-web.litertlm`         |
| `src/llm/loader.ts`           | `maxTokens`, `topK`, `temperature`      | `80`, `40`, `0.9`                          |
| `src/tts/speech.ts`           | per-Tomodachi `pitch`/`rate` variance   | `1.2–1.7` / `1.05–1.15`                    |
| `src/render/app.ts`           | `BASE_W` × `BASE_H` (canvas resolution) | `480 × 270`                                |
| `src/ai/needs.ts`             | decay rates per need                    | tuned for ~5 min to be visibly hungry       |

## Known limitations

- **LLM only on WebGPU-capable browsers.** Firefox ships WebGPU but with
  quirks; Chrome / Edge is the supported target. If the model fails to
  load, the game still plays — barks, TTS, and full simulation work without it.
- **Single-language barks** (English). The LLM system prompt is also English.
- **No mobile touch optimization** — desktop browser first. Layout will
  adapt but tap targets are sized for mouse.
- **No server sync** — saves are local to the browser profile. Use the
  **Export State** / **Import State** buttons in the editor (work in
  progress) if you want to move saves between devices.
